Implementasi Stored Procedure Tingkat Lanjut pada Database AKADEMIK

Praktikum Pertemuan 10

Pada praktikum pertemuan ke-10 mata kuliah Pemrograman SQL Lanjut, saya mempelajari materi mengenai Stored Procedure II yang membahas konsep stored procedure tingkat lanjut pada database AKADEMIK. Materi ini sangat penting dalam pengembangan sistem database karena stored procedure digunakan untuk menyimpan kumpulan query SQL di dalam database sehingga dapat dipanggil kembali secara berulang tanpa harus menulis query dari awal. Dengan menggunakan stored procedure, proses pengolahan data menjadi lebih cepat, lebih aman, dan lebih terstruktur.

Pada praktikum ini, materi yang dipelajari meliputi penggunaan multi-parameter, kontrol alur menggunakan IF ELSE, penggunaan parameter OUT, serta operasi DML seperti INSERT, UPDATE, dan DELETE di dalam procedure. Selain itu, praktikum juga menggunakan beberapa konsep lain seperti JOIN, GROUP BY, dan HAVING untuk mengolah data dari beberapa tabel sekaligus.

Pengertian Stored Procedure

Stored procedure adalah kumpulan query SQL yang disimpan di dalam database dan dapat dipanggil kembali kapan saja. Dengan stored procedure, programmer dapat membuat proses pengolahan data menjadi lebih efisien karena query tidak perlu ditulis berulang kali. Stored procedure juga membantu meningkatkan keamanan database karena pengguna tidak langsung mengakses tabel utama.

Multi Parameter

Materi pertama yang dipelajari adalah penggunaan beberapa parameter atau multi-parameter pada stored procedure. Parameter digunakan untuk menerima input dari pengguna sehingga procedure dapat menjalankan query berdasarkan kondisi tertentu. Dengan adanya multi-parameter, proses pencarian dan filter data menjadi lebih fleksibel. 

CREATE PROCEDURE filterMhs(
  IN p_prodi INT,
  IN p_thn INT
)
BEGIN
  SELECT * FROM mahasiswa
  WHERE idprodi = p_prodi
  AND thn_masuk = p_thn;
END;

Query tersebut digunakan untuk menampilkan data mahasiswa berdasarkan program studi dan tahun masuk tertentu. Dengan menggunakan parameter p_prodi dan p_thn, pengguna dapat mencari data sesuai kebutuhan tanpa harus mengubah isi query utama.

IF ELSE pada Stored Procedure

Selain multi-parameter, saya juga mempelajari penggunaan kontrol alur seperti IF, ELSEIF, dan ELSE di dalam stored procedure. Fitur ini memungkinkan procedure menjalankan perintah yang berbeda berdasarkan kondisi tertentu. 

IF p_mode = 1 THEN
  SELECT * FROM mahasiswa;
ELSE
  SELECT * FROM dosen;
END IF;

Dengan adanya IF ELSE, query dapat berjalan lebih dinamis dan menyesuaikan kondisi yang diberikan oleh pengguna.

OUT Parameter

Materi berikutnya adalah penggunaan parameter OUT. Parameter OUT digunakan untuk menghasilkan nilai keluaran dari procedure. Biasanya digunakan untuk mengambil hasil perhitungan atau agregasi data seperti jumlah mahasiswa, total nilai, dan lain sebagainya. 

CREATE PROCEDURE countMhs(
  IN p_prodi INT,
  OUT p_count INT
)
BEGIN
  SELECT COUNT(*) INTO p_count
  FROM mahasiswa
  WHERE idprodi = p_prodi;
END;

Pada query tersebut, procedure digunakan untuk menghitung jumlah mahasiswa berdasarkan program studi tertentu. Hasil perhitungan kemudian disimpan ke dalam variabel output p_count.

Operasi DML

Selain itu, saya juga mempelajari operasi DML di dalam stored procedure. DML atau Data Manipulation Language terdiri dari INSERT, UPDATE, dan DELETE yang digunakan untuk memanipulasi data pada tabel database. 

INSERT INTO mahasiswa(nama, idprodi)
VALUES('Rizqi', 1);

UPDATE mahasiswa
SET nama = 'Rizqi Akbar'
WHERE nim = '23312001';

DELETE FROM mahasiswa
WHERE nim = '23312001';

Dengan menyimpan operasi DML di dalam procedure, proses pengolahan data menjadi lebih praktis dan otomatis.

Kesimpulan

Awalnya saya cukup kesulitan memahami alur query di dalam procedure karena struktur penulisannya lebih kompleks dibanding query SQL biasa. Namun setelah mencoba langsung pada simulator SQL dan SSMS, saya mulai memahami bagaimana cara kerja stored procedure dengan lebih baik.

Dengan memahami stored procedure, saya menjadi lebih paham bagaimana cara membuat proses pengolahan data menjadi lebih otomatis, terstruktur, dan efisien.
