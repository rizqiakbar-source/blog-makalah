Implementasi Trigger pada Database AKADEMIK

Praktikum Pertemuan 11

Pada praktikum pertemuan ke-11 mata kuliah Pemrograman SQL Lanjut, saya mempelajari materi mengenai Trigger pada database AKADEMIK. Trigger merupakan kode SQL yang dijalankan secara otomatis ketika terjadi suatu event tertentu pada tabel database, seperti INSERT, UPDATE, dan DELETE. Materi ini sangat penting karena trigger sering digunakan untuk audit data, validasi tambahan, serta otomatisasi proses pada database.

Pada praktikum ini saya mempelajari konsep dasar trigger, jenis-jenis trigger, penggunaan BEFORE dan AFTER trigger, serta penggunaan referensi OLD dan NEW untuk membaca data sebelum dan sesudah perubahan terjadi. Selain itu, saya juga belajar bagaimana trigger dapat digunakan untuk membuat audit log dan validasi otomatis pada database.

Pengertian Trigger

Trigger adalah sekumpulan perintah SQL yang akan dijalankan secara otomatis ketika terjadi suatu event tertentu pada tabel database. Trigger bekerja tanpa harus dipanggil secara manual karena database akan langsung mengeksekusinya saat event terjadi.

Trigger biasanya digunakan untuk menjaga konsistensi data, membuat pencatatan aktivitas, validasi tambahan, serta otomatisasi proses tertentu pada sistem informasi berbasis database.

Jenis Trigger

Pada praktikum ini saya mempelajari beberapa jenis trigger yang umum digunakan pada database, yaitu:

  • BEFORE Trigger
  • AFTER Trigger
  • INSERT Trigger
  • UPDATE Trigger
  • DELETE Trigger

BEFORE Trigger dijalankan sebelum event terjadi, sedangkan AFTER Trigger dijalankan setelah event berhasil dilakukan.

BEFORE INSERT Trigger

Contoh trigger BEFORE INSERT digunakan untuk melakukan validasi data sebelum data masuk ke dalam tabel. 

CREATE TRIGGER cek_nilai
BEFORE INSERT ON nilai
FOR EACH ROW
BEGIN
  IF NEW.nilai < 0 THEN
    SIGNAL SQLSTATE '45000'
    SET MESSAGE_TEXT = 'Nilai tidak boleh negatif';
  END IF;
END;

Trigger tersebut digunakan untuk mencegah input nilai negatif pada tabel nilai. Jika pengguna memasukkan nilai kurang dari nol, maka database akan menampilkan pesan error.

AFTER INSERT Trigger

Selain BEFORE Trigger, saya juga mempelajari AFTER Trigger yang digunakan setelah event berhasil dijalankan. 

CREATE TRIGGER log_mahasiswa
AFTER INSERT ON mahasiswa
FOR EACH ROW
BEGIN
  INSERT INTO audit_log(keterangan)
  VALUES('Data mahasiswa baru ditambahkan');
END;

Trigger tersebut digunakan untuk mencatat aktivitas penambahan data mahasiswa ke dalam tabel audit_log secara otomatis.

Penggunaan OLD dan NEW

Pada trigger, OLD digunakan untuk mengambil data lama sebelum perubahan terjadi, sedangkan NEW digunakan untuk mengambil data baru setelah perubahan dilakukan. 

CREATE TRIGGER update_nama
BEFORE UPDATE ON mahasiswa
FOR EACH ROW
BEGIN
  INSERT INTO log_update
  VALUES(OLD.nama, NEW.nama);
END;

Dengan menggunakan OLD dan NEW, database dapat menyimpan riwayat perubahan data secara otomatis.

Keuntungan Trigger

Trigger memiliki banyak keuntungan dalam pengolahan database. Salah satunya adalah membantu otomatisasi proses tanpa harus menjalankan query secara manual. Selain itu, trigger juga membantu menjaga konsistensi data serta meningkatkan keamanan sistem database.

Trigger juga sangat berguna dalam pembuatan audit log karena setiap perubahan data dapat dicatat secara otomatis oleh sistem.

Penerapan pada Database AKADEMIK

Pada database AKADEMIK, trigger dapat digunakan untuk mencatat perubahan data mahasiswa, validasi input nilai, serta pencatatan aktivitas pengguna pada sistem akademik. Dengan adanya trigger, sistem informasi akademik menjadi lebih otomatis dan efisien.

Selain itu, trigger membantu mengurangi kesalahan input data karena validasi dilakukan langsung oleh server database.

Kesimpulan

Menurut saya, materi Trigger sangat menarik karena database dapat menjalankan proses secara otomatis ketika terjadi perubahan data. Awalnya saya cukup kesulitan memahami penggunaan OLD dan NEW, tetapi setelah mencoba langsung pada simulator SQL dan SSMS, saya mulai memahami cara kerja trigger dengan lebih baik.

Praktikum ini memberikan pengalaman baru mengenai otomatisasi database dan membantu saya memahami bagaimana sistem database bekerja secara lebih kompleks. Dengan memahami trigger, saya berharap dapat membuat sistem informasi berbasis database yang lebih aman, terstruktur, dan efisien di masa mendatang.
