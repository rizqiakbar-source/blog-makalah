Implementasi Stored Procedure Tingkat Lanjut pada Database AKADEMIK

Pada praktikum pertemuan ke-10 mata kuliah Pemrograman SQL Lanjut, saya mempelajari materi mengenai Stored Procedure II yang membahas konsep stored procedure tingkat lanjut pada database AKADEMIK.

Pengertian Stored Procedure

Stored procedure adalah kumpulan query SQL yang disimpan di dalam database dan dapat dipanggil kembali kapan saja. Dengan stored procedure, proses pengolahan data menjadi lebih cepat, aman, dan terstruktur.

Multi Parameter

Pada praktikum ini saya mempelajari penggunaan multi-parameter pada stored procedure. 

CREATE PROCEDURE filterMhs(
  IN p_prodi INT,
  IN p_thn INT
)
BEGIN
  SELECT * FROM mahasiswa
  WHERE idprodi = p_prodi
  AND thn_masuk = p_thn;
END;

IF ELSE

Saya juga mempelajari penggunaan IF ELSE untuk membuat logika kondisi pada stored procedure. 

IF p_mode = 1 THEN
  SELECT * FROM mahasiswa;
ELSE
  SELECT * FROM dosen;
END IF;

OUT Parameter

Parameter OUT digunakan untuk menghasilkan nilai keluaran dari procedure. 

CREATE PROCEDURE countMhs(
  IN p_prodi INT,
  OUT p_count INT
)
BEGIN
  SELECT COUNT(*) INTO p_count
  FROM mahasiswa
  WHERE idprodi = p_prodi;
END;

Kesimpulan
