Implementasi Stored Procedure pada Database AKADEMIK

Praktikum Pertemuan 9

Pada praktikum pertemuan ke-9 mata kuliah Pemrograman SQL Lanjut, saya mempelajari materi mengenai Stored Procedure pada database AKADEMIK. Stored procedure merupakan kumpulan perintah SQL yang disimpan di dalam database dan dapat dijalankan kembali kapan saja sesuai kebutuhan. Materi ini sangat penting karena stored procedure digunakan untuk mempermudah proses pengolahan data serta meningkatkan efisiensi dalam menjalankan query SQL.

Pada praktikum ini, saya mempelajari cara membuat stored procedure sederhana menggunakan parameter input dan menjalankan procedure tersebut untuk menampilkan data dari tabel mahasiswa. Selain itu, saya juga belajar bagaimana procedure dapat digunakan untuk mempermudah proses query yang sering digunakan secara berulang.

Pengertian Stored Procedure

Stored procedure adalah sekumpulan query SQL yang disimpan di dalam database server. Procedure ini dapat dipanggil kembali tanpa harus menulis ulang query SQL yang sama. Dengan menggunakan stored procedure, proses pengolahan data menjadi lebih cepat, aman, dan lebih mudah dikelola.

Stored procedure biasanya digunakan pada aplikasi berbasis database untuk mengatur proses pengambilan data, manipulasi data, maupun proses otomatis lainnya.

Membuat Stored Procedure

Pada praktikum ini saya mempelajari cara membuat stored procedure sederhana menggunakan parameter input. Contoh query yang digunakan adalah sebagai berikut: 

CREATE PROCEDURE tampilMahasiswa(
  IN p_prodi INT
)
BEGIN
  SELECT * FROM mahasiswa
  WHERE idprodi = p_prodi;
END;

Query tersebut digunakan untuk menampilkan data mahasiswa berdasarkan program studi tertentu. Parameter p_prodi digunakan sebagai input agar pengguna dapat memilih program studi yang ingin ditampilkan.

Menjalankan Stored Procedure

Setelah procedure berhasil dibuat, langkah berikutnya adalah menjalankan procedure menggunakan perintah CALL. 

CALL tampilMahasiswa(1);

Perintah tersebut digunakan untuk menampilkan data mahasiswa dengan id program studi 1. Dengan menggunakan procedure, query menjadi lebih sederhana dan mudah digunakan kembali.

Keuntungan Stored Procedure

Stored procedure memiliki banyak keuntungan dalam pengolahan database. Salah satunya adalah meningkatkan efisiensi karena query tidak perlu ditulis berulang kali. Selain itu, stored procedure juga membantu meningkatkan keamanan database karena pengguna tidak langsung mengakses tabel utama.

Keuntungan lainnya adalah proses maintenance menjadi lebih mudah. Jika terdapat perubahan query, programmer hanya perlu memperbarui procedure tanpa mengubah seluruh kode aplikasi.

Penerapan pada Database AKADEMIK

Pada database AKADEMIK, stored procedure dapat digunakan untuk menampilkan data mahasiswa, data dosen, data mata kuliah, maupun proses pengolahan nilai mahasiswa. Dengan adanya stored procedure, pengolahan data akademik menjadi lebih terstruktur dan efisien.

Selain itu, stored procedure juga membantu dalam membuat sistem informasi akademik yang lebih cepat dan stabil karena sebagian proses dijalankan langsung oleh server database.

Kesimpulan

Menurut saya, materi Stored Procedure sangat penting untuk dipelajari karena membantu memahami bagaimana database bekerja secara lebih kompleks. Awalnya saya sedikit kesulitan memahami sintaks pembuatan procedure, tetapi setelah mencoba langsung menggunakan simulator SQL dan SSMS, saya mulai memahami cara kerja stored procedure dengan lebih baik.

Praktikum ini memberikan pengalaman baru dalam pengolahan database karena saya belajar bagaimana membuat query yang lebih efisien, terstruktur, dan dapat digunakan kembali. Dengan memahami stored procedure, saya berharap dapat membuat aplikasi berbasis database yang lebih baik di masa mendatang.
